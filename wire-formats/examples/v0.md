# ORS Wire Format v0 Examples

See [../v0.md](../v0.md) for the full v0 specification.

## Payload Layout

Every v0 OP_RETURN payload has a fixed 101-byte header followed by kind-specific data:

```
[0:3]    magic   = 4f 52 53          ("ORS")
[3]      version = 00
[4:36]   pubkey  = <32 bytes x-only Schnorr pubkey>
[36:100] sig     = <64 bytes Schnorr signature over sha256(unsigned_payload)>
[100]    kind    = <1 byte kind>
[101:]   data    = <kind-specific bytes>
```

## Signing

```
unsigned_payload = magic || version || pubkey || kind || kind_data
message          = sha256(unsigned_payload)
sig              = schnorr_sign(privkey, message)
final_payload    = magic || version || pubkey || sig || kind || kind_data
```

The sig field (bytes 36–99) is excluded from the signed message. The signature binds the pubkey, kind, and content together.

## Test Vectors

### Invalid: Wrong Magic

```
OP_RETURN starts with: 4f534600...
Result: Not an ORS post (magic is "OSF" not "ORS")
```

### Invalid: Too Short

```
OP_RETURN: 4f525300 (only 4 bytes)
Result: Rejected — payload must be at least 102 bytes
```

### Invalid: Bad Signature

```
Header: valid magic/version/pubkey/kind, but sig does not verify against unsigned_payload
Result: Rejected — Schnorr signature invalid
```
