# ORS Wire Format v1 Examples

See [../v1.md](../v1.md) for the full v1 specification.

## Chunk Layout

### Chunk 0 (root)

```
[0:3]   magic        = 4f 52 53     ("ORS")
[3]     version      = 01
[4]     chunk_num    = 00
[5]     total_chunks = <number of chunks>
[6:80]  body_slice   = body[0:74]   (up to 74 bytes)
```

### Chunk N (N >= 1)

```
[0:3]   magic      = 4f 52 53       ("ORS")
[3]     version    = 01
[4]     chunk_num  = <N>
[5:80]  body_slice = body[74 + (N-1)*75 : 74 + N*75]  (up to 75 bytes)
```

### Body (distributed across chunks)

```
body = pubkey(32) || sig(64) || kind(1) || kind_data
```

## Signing

```
signing_body = pubkey(32) || kind(1) || kind_data
message      = sha256(signing_body)
sig          = schnorr_sign(privkey, message)
```

Note: the signing body excludes magic and version (unlike v0) to save 4 bytes across the chunk budget.

## Schematic 2-Chunk Example

A minimal v1 post with 52 bytes of kind_data (e.g. a short text note) fits in 2 chunks:

```
body = pubkey(32) || sig(64) || kind(1) || kind_data(52) = 149 bytes

Chunk 0 (80 bytes):
  4f 52 53 01 00 02  [body bytes 0..73]
  ^magic  ^v ^cn ^tc

Chunk 1 (80 bytes):
  4f 52 53 01 01  [body bytes 74..148]
  ^magic  ^v ^cn
```
